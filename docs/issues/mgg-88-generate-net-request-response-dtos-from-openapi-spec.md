# Generate .NET Request/Response DTOs from OpenAPI spec

## Context\\n\\nWith the OpenAPI spec established as the authoritative API contract (MGG-21), the .NET Request/Response DTOs should be **generated** from the spec — not hand-written. This replaces the current ViewModels (`AccountVM`, `ReceiptVM`, etc.) with generated, spec-driven DTOs.\\n\\n## Technology: NSwag.MSBuild or Kiota\\n\\n> **Important:** MGG-100 evaluates Kiota as a replacement for NSwag. That evaluation should complete BEFORE this issue starts to avoid migrating twice. The implementation details below assume NSwag — update if Kiota is selected.\\n\\n**Package:** `NSwag.MSBuild` 14.6.3+ (NuGet)\\n\\nNSwag can generate **server-side C# model classes** from an OpenAPI spec with MSBuild integration. Kiota (Microsoft's actively-maintained generator) is being evaluated as an alternative — DTOs are transport models identical on client and server, so "client" model generation produces the same POCOs the server needs.\\n\\n### Install\\n\\n`xml\n<!-- Directory.Packages.props -->\n<PackageVersion Include=\"NSwag.MSBuild\" Version=\"14.6.3\" />\n`\\n\\n`xml\n<!-- src/Presentation/API/API.csproj -->\n<PackageReference Include=\"NSwag.MSBuild\" Version=\"14.6.3\">\n  <PrivateAssets>all</PrivateAssets>\n  <IncludeAssets>runtime; build; native; contentfiles; analyzers</IncludeAssets>\n</PackageReference>\n`\\n\\n### NSwag Configuration (`nswag.json` at repo root)\\n\\n`json\n{\n  \"runtime\": \"Net90\",\n  \"documentGenerator\": {\n    \"fromDocument\": {\n      \"url\": \"openapi/spec.yaml\"\n    }\n  },\n  \"codeGenerators\": {\n    \"openApiToCSharpClient\": {\n      \"generateClientClasses\": false,\n      \"generateDtoTypes\": true,\n      \"namespace\": \"API.Generated.Dtos\",\n      \"output\": \"src/Presentation/API/Generated/Dtos.g.cs\",\n      \"dateType\": \"System.DateOnly\",\n      \"dateTimeType\": \"System.DateTimeOffset\",\n      \"generateNullableReferenceTypes\": true,\n      \"generateDataAnnotations\": true,\n      \"classStyle\": \"Record\"\n    }\n  }\n}\n`\\n\\n**Key settings:**\\n\\n\* `generateClientClasses: false` — we only want the DTO models, not an HTTP client\\n\* `generateDtoTypes: true` — generates the schema classes\\n\* `classStyle: \"Record\"` — generates C# records for immutability (matches Request/Response pattern)\\n\* `generateNullableReferenceTypes: true` — matches project-wide nullable enabled\\n\* `generateDataAnnotations: true` — generates `[Required]`, `[MinLength]`, `[Range]` etc. from spec constraints\\n\\n### Wire into MSBuild\\n\\n`xml\n<!-- src/Presentation/API/API.csproj -->\n<Target Name=\"NSwag\" BeforeTargets=\"CoreCompile\" Inputs=\"$(MSBuildProjectDirectory)/../../../openapi/spec.yaml\" Outputs=\"$(MSBuildProjectDirectory)/Generated/Dtos.g.cs\">\n  <Message Text=\"Regenerating DTOs from OpenAPI spec...\" Importance=\"high\" />\n  <Exec Command=\"$(NSwagExe_Net90) run $(MSBuildProjectDirectory)/../../../nswag.json\" />\n</Target>\n`\\n\\n**Important:** The `Inputs`/`Outputs` attributes enable incremental build — NSwag only runs when `spec.yaml` is newer than `Dtos.g.cs`. This prevents unnecessary regeneration on every build.\\n\\n**Note on .NET 10:** As of Feb 2026, NSwag may not yet have a `$(NSwagExe_Net100)` target. Use `$(NSwagExe_Net90)` — the generated code is runtime-agnostic. Monitor [NSwag#5216](<https://github.com/RicoSuter/NSwag/issues/5216>) for .NET 10 support.\\n\\n## Pre-commit Hook Integration\\n\\nThe pre-commit hook (defined in MGG-21) handles .NET DTO validation:\\n\\n`bash\n# Step 2 of .husky/pre-commit:\n# Regenerate DTOs and check for staleness\ndotnet build src/Presentation/API/API.csproj -t:NSwag --no-restore -v quiet\ngit diff --exit-code -- \"src/Presentation/API/Generated/\" \\\n  || (echo \"ERROR: Generated .NET DTOs are stale. Run 'dotnet build' and stage Generated/Dtos.g.cs\" && exit 1)\n`\\n\\n**What this catches:**\\n\\n\* Developer edits `spec.yaml` but forgets to rebuild → stale check fails\\n\* Developer hand-edits `Dtos.g.cs` → regeneration overwrites, stale check fails\\n\* LLM agent finds and directly edits the spec without rebuilding → stale check fails\\n\* Spec has breaking change → unit tests fail after regeneration\\n\\n## FluentValidation Strategy\\n\\nWith `generateDataAnnotations: true`, the generated DTOs include DataAnnotation attributes for constraints expressible in the spec (`[Required]`, `[MinLength]`, `[Range]`, `[RegularExpression]`, etc.). To avoid dual enforcement:\\n\\n\* **Spec-expressible constraints** (required, minLength, min/max, pattern) → rely solely on generated DataAnnotations\\n\* **Business rules only** (cross-field validation, database lookups, temporal logic) → FluentValidation\\n\* Remove any FluentValidation rules that duplicate spec constraints\\n\\nThis significantly reduces the FluentValidation surface and eliminates the risk of spec and validator diverging.\\n\\n## Naming Convention (enforced in spec)\\n\\n| Operation | Schema Name | ID Property | Generated C# Record |\\n| -- | -- | -- | -- |\\n| Create | `CreateAccountRequest` | **No ID** | `record CreateAccountRequest(string Name, ...)` |\\n| Read | `AccountResponse` | `Guid Id` (required) | `record AccountResponse(Guid Id, string Name, ...)` |\\n| Update | `UpdateAccountRequest` | `Guid Id` (required) | `record UpdateAccountRequest(Guid Id, string Name, ...)` |\\n\\n## Scope\\n\\nEntities: Account, Receipt, ReceiptItem, Transaction\\nAggregates: ReceiptWithItems, TransactionAccount, Trip\\n\\n## Tasks\\n\\n- \[ \] Evaluate Kiota vs NSwag (MGG-100 — must complete first)\\n- \[ \] Add chosen codegen tool to `Directory.Packages.props` and API.csproj\\n- \[ \] Create codegen config (e.g., `nswag.json`) with model-only generation\\n- \[ \] Add MSBuild `<Target>` with incremental build support (`Inputs`/`Outputs`)\\n- \[ \] Create `src/Presentation/API/Generated/` directory (commit generated file)\\n- \[ \] Add staleness check to pre-commit hook (`git diff --exit-code -- Generated/`)\\n- \[ \] Verify generated DTOs compile and have correct types\\n- \[ \] Update API controllers to use generated DTOs instead of ViewModels\\n- \[ \] Update Mapperly mappers: Domain ↔ generated DTOs\\n  -
